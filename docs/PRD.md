# PRD（Product Requirements Document）

## 1. 目的

「よすが式（数画式）」を、Web/PWAで "いつでもどこでも" 学習できる教材として提供する。

本質は「動画」ではなく「判断フロー（ライン→ナウキャスト→シナリオ→タイミング）を反復して身につける」こと。

## 2. ユーザー

- トレード判断を体系的に学びたい人
- 移動中やスキマ時間に学習したい人
- オフライン環境でも学習を継続したい人

## 3. 提供価値

- **反復学習による判断フローの習得**
  - 動画視聴ではなく、実際の判断を繰り返すことで身につける
  - 1問=1トレード判断として、実践的な学習を提供

- **いつでもどこでも学習可能**
  - PWA対応により、スマホにインストールしてオフラインでも利用可能
  - ネットワーク環境に依存しない学習体験

- **進捗の可視化（煽り指標なし）**
  - 正答率・勝率は表示しない
  - 反復量（回数）と継続（学習日数）のみを表示
  - 学習者を煽らず、継続を促す設計

## 4. スコープ（MVP）

### 必須機能

#### 4.1 学習カリキュラム閲覧
- Phase0: ライン（Line）
- Phase1: ナウキャスト（Nowcast）
- Phase2: シナリオ（Scenario）
- Phase3: タイミング（Timing）
- 各Phaseはテキスト＋図解の枠で表示
- 1画面=1テーマで構成

#### 4.2 問題集
- 1問ずつ出題（ランダム/次へ）
- 回答フロー：
  1. ライン有効？（Yes/No）
  2. ナウキャスト選択（例: Trend/Range/Transition 等）
  3. シナリオ成立？（Yes/No）
  4. タイミング来てる？（Yes/No）
  5. 結論：Entry / Skip
- 解説表示（なぜその判断順か）

#### 4.3 進捗管理
- 問題ごとの「解いた回数」「自分の選択」「メモ」を保存
- **正答率は表示しない**（重要）
- ローカルストレージに保存

#### 4.4 お気に入り
- 問題のブックマーク機能
- お気に入り一覧から直接問題にアクセス可能

#### 4.5 PWA（オフライン対応）
- 学習テキスト/問題データをキャッシュ
- ネット無しでも閲覧と回答ができる
- 同期機能はMVPでは不要

### 非対応（MVP外）

- 課金/決済機能
- アカウント機能
- サーバ同期機能
- 管理画面
- 動画ホスティング
- 正答率・勝率の表示
- 煽り指標（「もっと頑張ろう」等のメッセージ）

## 5. 成功指標（MVP）

- 学習ページが正常に表示される
- 問題を解いて回答フローを体験できる
- 進捗がローカルに保存される
- PWAとしてインストール可能
- オフラインで最低限の機能が動作する
