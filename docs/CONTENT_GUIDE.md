# CONTENT_GUIDE（コンテンツ作成ガイド）

## 1. 文章ルール

### 1.1 基本ルール

- **1文40文字以内**
  - 長文は避け、短く明確に
  - 読者が理解しやすい文章を心がける

- **抽象論NG、判断基準のみ**
  - 「〜が重要」「〜を意識する」などの抽象的な表現は避ける
  - 具体的な判断基準を記載する
  - 例：
    - ❌ 「ラインは重要です」
    - ✅ 「ラインは価格が3回以上タッチした場合に有効」

- **「〜と思う」「〜かもしれない」禁止**
  - 不確かな表現は使わない
  - 明確な判断基準として記載する
  - 例：
    - ❌ 「この場合はエントリーしても良いと思う」
    - ✅ 「この場合はエントリーする」

### 1.2 学習ページの構成

- **1画面=1テーマ**
  - 1つの学習ページは1つのテーマに絞る
  - 複数のテーマを混在させない

- **見出し構造**
  - `#` タイトル（Phase名）
  - `##` セクション（例: 「ラインとは」）
  - `###` サブセクション（例: 「有効ラインの条件」）

- **図解の扱い**
  - Markdown内で画像を参照可能
  - 画像は `/public/images/` に配置
  - 図解には必ず説明文を付ける

### 1.3 問題の構成

- **1問=1判断**
  - 1つの問題は1つのトレード判断を扱う
  - 複数の判断を同時に問う問題は作らない

- **問題の流れ**
  1. チャート画像（任意）
  2. 回答フロー（5ステップ）
  3. 解説

- **解説の書き方**
  - 各ステップの判断理由を明確に
  - 「なぜその判断順か」を説明
  - 抽象論ではなく、具体的な判断基準を記載

## 2. 学習ページのテンプレート

```markdown
# Phase0: ライン

## ラインとは

ラインは価格の支持線・抵抗線のこと。
3回以上タッチしたラインが有効。

## 有効ラインの条件

1. 価格が3回以上タッチ
2. タッチ時の反発が明確
3. 時間軸が適切

## ラインの見方

チャート上で価格が反発した点を結ぶ。
水平線または斜め線で描画する。
```

## 4. 解説の書き方の例

### 良い解説

```
ラインは価格が3回以上タッチしているため有効。
ナウキャストは上昇トレンドが継続している。
シナリオは上昇シナリオが成立している。
タイミングはライン付近で反発しているため良好。
結論としてエントリー条件を満たしている。
```

### 悪い解説

```
ラインは重要だと思います。
ナウキャストはトレンドかもしれません。
シナリオは成立していると思う。
タイミングも良いかもしれない。
エントリーしても良いでしょう。
```
